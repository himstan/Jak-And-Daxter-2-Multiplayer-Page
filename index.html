Before playing

A few things to check first

1

Use the same version

Both players must use the same version of the game.

Jak hosts

The Host player will be playing as Jak, so keep that in mind before you set up a session.

Daxter joins

The second player will be playing as Daxter, who can do pretty much everything the Jak can. (Other than turn into Dark Jak :c)

2

Online play may need setup

The Host may need to manually forward the default 26210 port if they have UPnP disabled on their router.

Jak · Host

Host a game

Host the game, copy the invite token, wait for a player to connect.

  1. 1

    Choose Host Game

    Start the mod and select Host Game from the title menu.

  2. 2

    Choose Host LAN or Host Online

    Select Host LAN if playing locally, or Host Online to play online.

  3. 3

    Choose New Game or Load Game

    Choose New Game for a fresh run, or Load Game to continue an existing save.

  4. 4

    Copy and send invite

    Select Copy Invite and send it privately to the other player. Wait on the host screen until they connect.

    For online play, the Host may need to forward the default 26210 port in their router.

    If the Copy Invite stays red, then it probably means that the automatic port mapping has failed, and the 26210 port is not forwarded. I suggest using a VPN solution like Hamachi or Tailscale to fake LAN play. Host LAN and join Direct to their assigned IP address. You don't need an invite token for LAN play.

  5. 5

    Start playing

    Once the client player connects, the game will automatically continue into the selected save or new game.

Daxter · Join

Join a game

Choose between connecting via invite token (recommended for online), scanning the local network, or direct manual connection.

  1. 1

    Choose Join Game

    Start the same mod version as the host and select Join Game from the title menu.

  2. 2

    Option 1: With Invite Token

    Copy the host's invite string to your clipboard. Choose With Invite Token, select Paste Invite, and select Connect once the invite is ready.

  3. 3

    Option 2: Scan LAN

    When playing on the same local network (LAN) or local split-screen, choose Scan LAN. The game will automatically search for active hosts and connect.

  4. 4

    Option 3: Direct Connect

    (Note: All fields support CTRL+V paste)
    To connect manually, choose Direct Connect, verify or update the target Address and Port, then select Connect.

    The Token is not used for LAN play.
    So if you join your friend using their Hamachi/Tailscale Ip, you just copy their ip and paste it in here, and you should be set!

Using an Invite

Online invite token

Online invites use the format: address:port/token. Do not post an invite publicly as it contains the host's public IP address! It is a temporary credential for the current hosting session.

Once connected, you will join the game session as Daxter!

Jak
Daxter

Play "split-screen"

Run two windowed game instances side by side on the same monitor. Host in one window, then choose Join Game → Scan LAN in the other.

Each instance should recognize a connected controller. With only one controller, use the controller for one window and the keyboard for the other.

Troubleshooting

Quick fixes

Anything looks desynced or broken on the client side?

The Reconnect button is going to act as your primary safeline while playing this mod, so don't be afraid to use it!

The client cannot find the host

  • Make sure both players are using the same version of the mod.
  • Check Windows Firewall on the host machine and allow the game/OpenGOAL through it.
  • For online play, the host should forward the UDP port 26210 on their router.
  • If all fails, then try Hamachi or Tailscale or any VPN solution, and play "LAN"!

The session ended

  • If the host leaves, the session is over.
  • Start hosting again and have the client reconnect.

Current build

Known limitations

These issues are expected in the first release and do not necessarily mean the connection was set up incorrectly.

Random crashes

There's a couple of random crashes, so expect that, especially on the non-host Daxter side.

Remote animations

A remote player can miss an animation trigger. For example, a jump may look like a falling animation until the player lands.

Traffic handover

Traffic is mainly controlled by the host. When Daxter moves outside Jak's traffic radius, the client may briefly remove traffic before spawning it locally.

Client-side soft-locks

Reconnect is currently the main recovery option when the client enters a broken state.

Limited testing

The mod has not been tested in every possible situation. Reproducible bugs and game-breaking issues can be reported on GitHub.
